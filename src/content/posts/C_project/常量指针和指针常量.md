要区分这三种指针，关键在于理解 `const` 关键字在声明中的位置。一个简单有效的记忆口诀是：**“左定值，右定向”**。

这里的“左”和“右”是相对于指针符号 `*` 而言的。

*   **左定值**：`const` 在 `*` 的左边，表示指针指向的**值**是常量，不可通过该指针修改。
*   **右定向**：`const` 在 `*` 的右边，表示指针本身的**地址**（即指向）是常量，不可改变其指向。

下面我们来详细解析这三种情况。

### 🎯 指向常量的指针 (Pointer to Constant)

这种指针的特点是：**指针的指向可以改变，但指针指向的值不能通过它来修改**。

*   **语法形式**：`const int *p;` 或 `int const *p;`
*   **解读**：`const` 在 `*` 的左边，所以是“**定值**”。你可以通过这个指针读取变量的值，但不能修改它。不过，你可以让这个指针指向另一个变量。

```c
int a = 10, b = 20;
const int *p = &a; // p 指向 a

*p = 100; // ❌ 错误！不能通过 p 修改 a 的值
p = &b;   // ✅ 正确！可以让 p 指向新的变量 b
```

### 🔒 指针常量 (Constant Pointer)

这种指针的特点是：**指针一旦初始化，其指向就不能再改变，但可以通过它修改所指向变量的值**。

*   **语法形式**：`int *const p;`
*   **解读**：`const` 在 `*` 的右边，所以是“**定向**”。指针 `p` 本身就像一个常量，必须在声明时初始化，并且之后不能再指向别的地址。但你可以通过它去修改它所指向的那个变量的值。

```c
int a = 10, b = 20;
int *const p = &a; // p 是一个常量，必须初始化，并固定指向 a

*p = 100; // ✅ 正确！可以通过 p 修改 a 的值
p = &b;   // ❌ 错误！p 的指向不能改变
```

### 🔐 指向常量的常量指针 (Constant Pointer to Constant)

这是最严格的一种，结合了前两者的特点：**指针的指向不能改变，也不能通过它修改所指向的值**。

*   **语法形式**：`const int *const p;`
*   **解读**：`*` 的左右两边都有 `const`，意味着“**定值**”又“**定向**”。指针的指向和指向的值都不能被修改。

```c
int a = 10, b = 20;
const int *const p = &a; // p 的指向和指向的值都被固定

*p = 100; // ❌ 错误！不能修改指向的值
p = &b;   // ❌ 错误！不能修改指针的指向
```

### 📊 总结对比

为了方便你快速回顾，这里有一个清晰的对比表格：

| 类型            | 语法示例                  | 能否修改指针指向？ | 能否通过指针修改值？ | 核心口诀 |
| :------------ | :-------------------- | :-------- | :--------- | :--- |
| **指向常量的指针**   | `const int *p;`       | ✅ 可以      | ❌ 不可以      | 左定值  |
| **指针常量**      | `int *const p;`       | ❌ 不可以     | ✅ 可以       | 右定向  |
| **指向常量的常量指针** | `const int *const p;` | ❌ 不可以     | ❌ 不可以      | 左右都定 |
