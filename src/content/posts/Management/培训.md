## ubuntu安装方案对比
| 安装方案 | 优点       | 缺点               |
| :--: | -------- | ---------------- |
| WSL  | 方便       | 对Windows版本有硬性要求  |

## 深度学习的准备
自行上网搜索安装cuda+cudnn+pytorch，建议装在conda的虚拟环境中
以下是我踩过的坑
* 不要用任何源下载，下载慢的话可以用迅雷或者魔法上网
* conda记得设置安装路径和环境变量
* 注意显卡版本和cuda，cudnn的版本对照
* 注意cuda和pytorch的版本对照
* 注意pytorch和python的版本对照
安装完pytorch后运行以下代码
```python
import torch
print(torch.__version__)  # 查看torch当前版本号
#如果输出xxx+cu或者没有cu则下载成功，+ cpu建议再回头下载
print(torch.version.cuda)  # 编译当前版本的torch使用的cuda版本号
#如果输出版本号则下载成功，输出None建议回头再下载
print(torch.cuda.is_available())  # 查看当前cuda是否可用于当前版本的Torch
#如果输出True，则表示可用，输出False建议回头再下载
```

## 代理
* ### git代理
在git clone [Github](www.github.com)上的代码时有时会超时，就需要挂代理。
启动代理运行`git config --global http.proxy "http://127.0.0.1:7890"`和`git config --global https.proxy "http://127.0.0.1:7890"`取消代理运行`git config --global --unset http.proxy`和`git config --global --unset https.proxy·
* ### requirements.txt代理
下载python包时有时候非常慢，可以走镜像源或者代理。
如原指令为pip install -r requirements.txt ，下载缓慢。
代理:pip install -r requirements.txt --proxy=127.0.0.1:7890
镜像源:pip install -r requirements.txt -i  https://pypi.mirrors.ustc.edu.cn/simple/
* ### CMakeList.txt代理
使用Cmake配置时会因为无法下载报大片红色error
在文件开头添加`set(ENV{http_proxy} "http://127.0.0.1:7890")``set(ENV{https_proxy} "http://127.0.0.1:7890")`即可加速下载
#### 其他软件代理可以自行百度`xxx走代理`

```cpp
#include <opencv2/opencv.hpp>
#include <iostream>
//需要在当前目录准备一个input.mp4
int main() {
    // 打开当前目录下的 input.mp4 视频文件
    cv::VideoCapture cap("input.mp4");
    // 检查是否成功打开视频
    if (!cap.isOpened()) {
        std::cerr << "无法打开视频文件 input.mp4" << std::endl;
        return -1;
    }  
    cv::namedWindow("Video", cv::WINDOW_AUTOSIZE);  
    cv::Mat frame;  
    // 逐帧读取并显示
    while (true) {
        cap >> frame; // 读取一帧  
        if (frame.empty()) break; // 如果帧为空，说明视频结束  
        cv::imshow("Video", frame); // 显示帧  
        // 按下 ESC 键退出播放
        if (cv::waitKey(30) == 27) {
            break;
       }
    }  
    // 清理资源
    cap.release();
    cv::destroyAllWindows();  
    return 0;
}
```
