# 哨兵技术特点
## 1. 多传感器融合 SLAM（激光-惯性里程计）

  

基于 HKU-MARS 的 **FAST-LIO2** 框架，采用紧耦合迭代扩展卡尔曼滤波（IEKF），将 LiDAR 点云与 IMU 数据深度融合。
  
```

FAST_LIO 管线：

  LiDAR 点云 ──► 预处理 ──► ikd-Tree 搜索 ──► IEKF 状态估计 ──► 增量建图 ──► 里程计输出

  IMU 数据   ──► 预积分 ──┘

```

  
## 2. 深度学习 + 传统视觉混合检测

  
采用 **YOLO 粗检 + 传统视觉精检** 的双阶段级联检测架构，兼顾速度与精度。

- **第一阶段（YOLO）**：使用 YOLOv8 模型对整帧图像进行目标检测，快速定位候选装甲板区域（ROI），利用 GPU加速推理
- **第二阶段（传统视觉）**：在 ROI 内进行自适应阈值二值化、形态学腐蚀/膨胀、边缘检测和轮廓提取，精确定位灯条的四个角点
- **灯条角点排序**：对提取的灯条角点按空间位置（左上、右上、左下、右下）进行一致性排序，确保后续 PnP 解算的输入正确

```

相机帧 → YOLO 推理 (GPU) → 装甲板 ROI → 传统视觉提取灯条 → 灯条角点排序 → 输出 4 个特征点

```

  

## 3. 卡尔曼滤波预测跟踪

  

基于 OpenCV 实现的 4 状态（x, y, vx, vy）卡尔曼滤波器，对目标运动进行预测。

- **双模式切换**：支持"跟踪原始目标点"和"跟踪预测目标点"两种模式，适应不同对抗场景
- **连续帧关联**：利用 IoU 计算相邻帧检测框的重合度，优先跟踪与上一帧重合的目标，解决多目标干扰问题
- **目标丢失处理**：丢失超过 25 帧后自动重置目标状态，避免误跟踪

  

## 4. PnP 三维位姿解算

  

通过 EPNP 算法（`cv2.solvePnP`）从 4 组 2D-3D 对应点解算目标的 3D 位姿。

- **输入**：4 个灯条角点的像素坐标（2D）+ 装甲板物理尺寸（3D，单位：cm）
- **输出**：Pitch 角、Yaw 角、目标距离，经过标定补偿后直接用于云台控制
- **相机标定**：预置高精度相机内参矩阵和畸变系数（`DEFAULT_CAMERA_MATRIX` / `DEFAULT_DIST_COEFFS`）  

```

4 个灯条角点 (2D) + 装甲板模型 (3D) ──► solvePnP ──► rvec, tvec ──► Pitch/Yaw/Distance

```
  
## 5. 有限状态机战术决策


基于 **血量 + 标志位** 驱动的有限状态机，实现自主战术决策。

| 状态 | 触发条件 | 行为 | 退出条件 |
|------|----------|------|----------|
| 状态 0 | 开局 (flag=4) | 导航至目标点抢占位置 | 到达目标点 |
| 状态 1 | 抢占完成 | 原地自瞄射击 | 血量 < 320 |
| 状态 2 | 血量不足 | 撤退至安全位置 | 到达安全点 |
| 状态 3 | 撤退完成 | 自瞄防守 | 血量 < 240 |
| 状态 4 | 血量过低 | 导航至补给点恢复 | 血量 > 390 → 回到状态 2 |

```
状态转移图：
  [0] 开局抢点 → [1] 自瞄攻击 → [2] 撤退 → [3] 防守 → [4] 补给 → 循环
```

（次年比赛采用了行为树的方案）

## 6. 串口自定义帧协议通信

  

与下位机（底盘/云台 MCU）通过串口进行自定义帧协议通信。

  

- **帧格式**：`0xFF | ... 数据 ... | 0xFE`，固定 15 字节帧长

- **传输内容**：标志位（flag）、血量（HP）、战术控制指令

- **双向通信**：上位机下发导航/云台指令，下位机上报状态和血量

  

## 7. 多模态定位方案


根据不同场景需求，提供多种定位方案灵活切换：

| 定位方式 | 适用场景 | 依赖 |
|---------|----------|------|
| FAST_LIO 里程计 | 实时建图/无先验地图 | LiDAR + IMU |
| HDL 全局定位 | 已知地图中的重定位 | 预建 PCD 地图 + LiDAR |
| AMCL 粒子滤波 | 2D 栅格地图导航 | 激光扫描 + 2D 地图 |


## 8. 点云预处理管线

完整的点云到导航地图的处理流水线：


```

3D 点云 (PCD) ──► 体素滤波 (VoxelGrid) 降采样 ──► 半径滤波去噪
                    │
                    ├──► PCD → PGM 栅格地图转换 (pcd2pgm)
                    │         └──► AMCL 导航定位
                    │
                    └──► 3D 点云 → 2D 激光扫描 (pointcloud_to_laserscan)
                              └──► move_base 局部路径规划

```

  

## 9. CUDA 加速点云配准

**fast_gicp** 模块提供 GPU 加速的 GICP/NDT 点云配准算法，大幅提升大规模点云配准速度。

- 支持 CUDA 加速的暴力 KNN 搜索

- 同时提供 CPU 和 CUDA 两种后端，灵活适配不同硬件

## 10. ROS 模块化系统架构

项目采用 ROS 模块化设计，各功能包独立编译、独立运行，通过 launch 文件灵活组合：

```
导航模式 (naving.launch)：
  ├── locating.launch        # 定位：FAST_LIO + HDL + Rviz
  ├── pcd2pgm                # 地图转换
  ├── pointcloud_to_laserscan # 点云转激光
  ├── move_base              # 全局/局部路径规划 + 代价地图
  ├── test_node              # 战术决策与串口通信
  └── YOLO (shooting.py)     # 视觉检测与目标跟踪
```

## 11. 海康工业相机深度集成
直接调用海康工业相机 SDK（MvCameraControl），实现低延迟图像采集：
- 设备枚举、GigE/USB 双协议支持
- 手动曝光/增益控制，适应不同光照条件
- 零拷贝获取图像帧，减少延迟
- 图像旋转 180° 校正相机安装方向
