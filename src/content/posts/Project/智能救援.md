此项目是一个基于 **OpenCV** 的计算机视觉项目，主要用于通过摄像头实时检测**彩色小球**（红、黄、蓝、黑）和**线条**（横向/纵向）。

----
## 项目特色
做了多种方案以及可控的参数供用户选择，小白也可快速上手

### 检测方案
项目同时实现了三种不同的球体检测方案，可根据场景灵活选用：

| 方案            | 文件                      | 原理                            | 适用场景           |
| ------------- | ----------------------- | ----------------------------- | -------------- |
| HSV 颜色掩码 + 轮廓 | `main.py`               | 基于 HSV 色彩空间做颜色范围筛选，再通过轮廓圆形度过滤 | 颜色特征明显、光照可控的场景 |
| 霍夫圆 + 颜色分类    | `find_ball_hough.py`    | 先检测圆形，再对圆心区域取色分类              | 球体形状规则、颜色多样的场景 |
| 模板匹配          | `find_ball_template.py` | 灰度模板滑动匹配                      | 目标外观固定的场景      |

### 相机参数的可编程校准
通过 OpenCV 的 `cap.set()` 直接控制相机硬件参数，在软件层面锁定光照条件，减少环境光变化对检测效果的影响。
```python

cap.set(cv2.CAP_PROP_EXPOSURE, -6)      # 曝光
cap.set(cv2.CAP_PROP_HUE, -5)           # 色调
cap.set(cv2.CAP_PROP_SATURATION, 60)    # 饱和度
cap.set(cv2.CAP_PROP_GAIN, -1)          # 增益
cap.set(cv2.CAP_PROP_CONTRAST, 0)       # 对比度
```

### 完整的数据标注流水线
```

make_datasets.py  →  LabelMe 标注  →  json2txt.py  →  convert_txt.py  →  split_dataset.py

（采集图像）          （人工标注）      （JSON→TXT）     （标签数字化）      （7:3 划分）

```

配套的格式修正工具：
- `negger.py`: 修复因坐标转换产生的负值
- `lim.py`: 统一小数精度到 6 位
**亮点**: 这套流水线覆盖了从数据采集、标注、格式转换、数据清洗到数据集划分的完整链路，是一个可复用的数据工程框架。

### 代码复用与模块化设计
多个检测模块共享一致的接口模式：
```python
# 统一的相机初始化 + 主循环模板
cap = cv2.VideoCapture(1)
cap.set(cv2.CAP_PROP_EXPOSURE, new_exposure)
while True:
    ret, frame = cap.read()
    # 处理逻辑
    cv2.imshow('window', frame)
    if key == ord('q'): break
    elif key == ord('s'): save_frame()
cap.release()
```
- `find_ball_hough.py` 复用 `find_line.py` 的 `my_canny()` 函数
- 各模块独立可运行，又可组合使用
