本项目是一个基于 **OpenCV** 的视觉识别系统，主要用于机器人抓取任务的视觉引导。系统通过摄像头采集图像，识别颜色、圆环、二维码、角点等特征，并通过串口与下位机通信完成任务调度。
系统在第十五届工训赛中稳定运行，获全国特等奖 。
本项目有以下特点
## 双重霍夫圆变换
**思路**：标准的霍夫圆变换参数难以同时兼顾"检出率"和"准确率"——`param2` 设置过小会引入大量误检，设置过大又容易漏检。本系统采用**两级级联策略**解决此问题。
**实现细节**：
```python
# 第一级：粗检测（低阈值，确保不漏检）
circles_pre = cv2.HoughCircles(grayImg, cv2.HOUGH_GRADIENT, 1, 100,
                               param1=150, param2=PARAM2,  # PARAM2=100
                                minRadius=r_min, maxRadius=r_max)
# 记录粗检测的粗略坐标
for (x, y, r) in circles_pre:
    pre_list.append((x, y, r))
# 第二级：精检测（高精度定位）
circles = cv2.HoughCircles(grayImg, cv2.HOUGH_GRADIENT, 1, 3,  # minDist=
                            param1=150, param2=PARAM2,
                            minRadius=10, maxRadius=70)
# 将精检测结果与粗检测坐标匹配聚类，取平均
for i in range(circle_size):
    x_avg = sum(x for x, y, r in circles_list[i]) / size
    y_avg = sum(y for x, y, r in circles_list[i]) / size
    res_list.append((x_avg, y_avg))
``` 
`DiscriminateRing_Pro()` 更进一步，将第二级检测切换到上一帧图像上进行，利用帧间差异消除当前帧的噪声干扰，实现**跨帧协同检测**。

## 基于 LAB 颜色空间的抗光照颜色识别
**思路**：RGB 颜色空间对光照变化极其敏感——同一种颜色在不同光照下 RGB 值差异巨大。而 LAB 颜色空间将亮度（L通道）与颜色信息（a、b通道）分离，从根本上隔离光照干扰。
在次年的工训赛中，更是引入了6色识别，利用LAB空间和标定，可以很快地解决颜色识别问题。
**关键设计**：直接取 ROI 区域内 a、b 通道的**均值**而非逐个像素判断，兼顾了效率和抗噪性。配合 `cv2.GaussianBlur(roi, (9, 9), 0)` 的预处理，进一步抑制了噪点影响。同时，蓝色判断阈值为 `a > 108` 而非 `a > 128`，说明实际调试中蓝色 a 通道值偏低，代码经过了**实际标定调优**。

## 缓冲区累积确认机制
**思路**：转盘旋转时因为抖动，光线波动出现位置偏移，单帧检测不可靠，采用**时空累积投票**策略--连续多帧检测到同一位置才确认，类似"去抖动"算法。
**实现细节**：
```python
RING_BUFFER = np.zeros((640, 480-TOP_BORDER), np.uint32)    # 像素级累积
RING_CHECK = np.zeros((64, 48-TOP_BORDER//10), np.uint32)   # 区域级累积 
def ZhuanPanConfirm(mat):
    # 每次检测到圆环，在对应位置计数+1
    RING_CHECK[int(ring[0]/10)][int(ring[1]/10)] += 1
    RING_BUFFER[int(ring[0])][int(ring[1])] += 1
    # 区域累积超过10次才确认
    if RING_CHECK[int(ring[0]/10)][int(ring[1]/10)] > 10:
        t = np.where(RING_BUFFER == np.max(RING_BUFFER))
        return x, y  # 返回累积次数最多的精确坐标
```
  

先粗筛位置（RING_CHECK 超过10次），再取精确位置（RING_BUFFER 中累计最多的坐标），兼顾了**判断速度**和**输出精度**。相比简单的帧间平滑滤波，这种方法对间歇性误检有更好的免疫力。

## 串口驱动的事件型任务调度架构
**思路**：系统架构采用"外部事件驱动"模式——下位机通过串口发送一个字节的任务码，上位机自动切换执行对应视觉任务，实现了一个**极简状态机**。
**实现细节**：
```python
# uart.py - 后台线程持续监听，被动接收任务码
def Uart_function(uart: Uart):
    while(uart.close_flag):
        buffer = uart.S4_Uart.read()
        uart.Task_id = buffer[-1]  # 收到即更新，无需轮询
# task.py - 主循环根据任务码分发
def Task(mat, u, w):
    if u.Task_id == 0x01:
        # 二维码扫描
    elif u.Task_id == 0x05:
        # 三色检测
    elif u.Task_id == 0x12:
        # 角点检测
    # ...
    # 任务完成后自动归零
    u.Task_id = 0x00
```

**架构特点**：
- 下位机通过串口**推送**任务码，上位机**无需轮询**，串口线程自动更新
- 任务函数执行完毕后将 `Task_id` 归零，隐式表达"任务完成"
- 任务码采用16进制编码（`0x01`~`0xC6`），同类任务通过高位/低位编码实现分组

这种设计在资源受限的嵌入式场景下非常高效——省去了自定义协议栈的复杂性，用一个字节就完成了任务调度。

## 矩形霍夫直线四级处理管线
**思路**：检测绿色矩形的中心点，采用"颜色分割 → 直线检测 → 分类 → 求交点 → 算中心"的五步管线，将矩形检测转化为直线检测问题。
**实现细节**：
```python
# 第一级：LAB颜色空间创建绿色掩码
mask = cv2.inRange(lab, lower_green, upper_green)  # L:78-246, A:101-160, B:147-180

# 第二级：形态学去噪
mask = cv2.morphologyEx(mask, cv2.MORPH_OPEN, kernel)   # 开运算去噪点
mask = cv2.morphologyEx(mask, cv2.MORPH_CLOSE, kernel)  # 闭运算填孔洞

# 第三级：霍夫直线检测 + 按角度分类
lines = cv2.HoughLines(edges, 2, np.pi/180, threshold=80)
if angle < 30 or angle > 150:   # 水平线
    horizontal_lines.append((rho, theta))
elif 60 < angle < 120:          # 垂直线
    vertical_lines.append((rho, theta))

# 第四级：计算交点 + 去重 + 算中心
corners = find_line_intersections(horizontal_lines, vertical_lines)
corners = remove_duplicate_corners(corners, min_distance=20)
center_x, center_y = calculate_rectangle_center(corners)
```


**去重机制**：同一角点可能被多组直线交点命中，通过 `remove_duplicate_corners()` 设置了20像素的最小距离阈值，确保每个角点只输出一次。
